## Usage

```tsx
import { ToolEditor } from "@/components/commandly/tool-editor/tool-editor";
```

### Basic Usage

```tsx
<ToolEditor tool={myTool} />
```

The editor initialises with the provided tool. Users can add commands and parameters interactively via the UI.

### With an Existing Tool

Pass your tool directly via the `tool` prop:

```tsx
<ToolEditor tool={myExistingTool} />
```

### Listening to Changes

Use the `onSave` callback to receive the updated tool when the user saves:

```tsx
function ToolPreview() {
  const [savedTool, setSavedTool] = useState<Tool | null>(null);

  return (
    <>
      <ToolEditor
        tool={myTool}
        onSave={(tool) => setSavedTool(tool)}
      />
      {savedTool && <pre>{JSON.stringify(savedTool, null, 2)}</pre>}
    </>
  );
}
```

## API Reference

### ToolEditor

The root component.

| Prop                   | Type                           | Default | Description                                                      |
| ---------------------- | ------------------------------ | ------- | ---------------------------------------------------------------- |
| `tool`                 | `Tool`                         | —       | The tool definition to initialise the editor with                |
| `onSave`               | `(tool: Tool) => void`         | —       | Called when the user saves the tool                              |
| `isNewTool`            | `boolean`                      | `false` | Indicates the tool has not been persisted yet                    |
| `savedCommands`        | `SavedCommand[]`               | `[]`    | Previously saved commands to display in the saved commands panel |
| `onSaveCommand`        | `(command: string) => void`    | —       | Called when the user saves a new command                         |
| `onDeleteSavedCommand` | `(commandKey: string) => void` | —       | Called when the user deletes a saved command                     |

### useToolBuilder

A context hook available to any component rendered inside `ToolEditor` (or inside your own `ToolBuilderProvider`).

```tsx
import { useToolBuilder } from "@/components/commandly/tool-editor/tool-editor.context";

function ToolStats() {
  const { tool } = useToolBuilder();
  return <span>{tool.parameters.length} parameters</span>;
}
```

| Value                          | Type                                       | Description                              |
| ------------------------------ | ------------------------------------------ | ---------------------------------------- |
| `tool`                         | `Tool`                                     | The current tool definition              |
| `selectedCommand`              | `Command`                                  | The currently focused command            |
| `selectedParameter`            | `Parameter \| null`                        | The currently focused parameter          |
| `parameterValues`              | `Record<string, ParameterValue>`           | Current parameter input values           |
| `updateTool`                   | `(updates: Partial<Tool>) => void`         | Update top-level tool fields             |
| `addSubcommand`                | `(parentKey?: string) => string`           | Add a new subcommand; returns its key    |
| `deleteCommand`                | `(commandKey: string) => void`             | Delete a command and all its subcommands |
| `upsertParameter`              | `(parameter: Parameter) => void`           | Add or update a parameter                |
| `removeParameter`              | `(parameterKey: string) => void`           | Remove a parameter                       |
| `getParametersForCommand`      | `(commandKey: string) => Parameter[]`      | Get non-global parameters for a command  |
| `getGlobalParameters`          | `() => Parameter[]`                        | Get all global parameters                |
| `getExclusionGroupsForCommand` | `(commandKey: string) => ExclusionGroup[]` | Get exclusion groups for a command       |

### Sub-components

| Component       | Description                                                                     |
| --------------- | ------------------------------------------------------------------------------- |
| `CommandTree`   | Displays commands in a hierarchical tree; click to select                       |
| `ParameterList` | Lists parameters for the selected command with add/edit/delete actions          |
| `PreviewTabs`   | Live preview tabs showing the rendered form, generated command, and JSON output |
| `HelpMenu`      | Reference panel for parameter types and flags                                   |

## Notes

- `ToolEditor` internally wraps everything in `ToolBuilderProvider` — use `onSave` to receive the updated tool, or `useToolBuilder()` inside child components
- Commands support unlimited nesting via `parentCommandKey`
- The exclusion groups dialog lets you define mutually exclusive parameter sets
- Saved commands management is handled via the `savedCommands`, `onSaveCommand`, and `onDeleteSavedCommand` props
- The block includes all dialogs (`CommandDialog`, `ParameterDetailsDialog`, `ToolDetailsDialog`, `ExclusionGroupsDialog`, `SavedCommandsDialog`)
